# Подход к ранжированию фасадов: Learning to Rank с Логистической Регрессией

## Обзор

Задача: присвоить каждому зданию числовой **Damage Score** по каждому из 5 классов повреждений,
отражающий относительную степень ущерба в сравнении с другими зданиями выборки.

Подход: **Learning to Rank** — обучение на попарных сравнениях, размеченных человеком-аннотатором.
Итоговая модель — Логистическая Регрессия на разности признаков, математически эквивалентная
линейной версии RankNet.

---

## 1. Признаковое пространство

### 1.1 Источник данных

SegFormer (Vision Transformer для семантической сегментации) обрабатывает каждое здание
по тайлам 800×800 пикселей. Для каждого тайла и каждого из 5 классов повреждений
(`coating_deterioration`, `cracks`, `masonry_degradation`, `moisture_bio_damage`, `vandalism`)
вычисляется **ratio** — доля пикселей тайла, относящихся к данному классу.

С переходом к мягким предсказаниям (softmax-порог вместо argmax):

$$\text{ratio}_{c,t} = \frac{1}{H \cdot W} \sum_{i,j} \mathbf{1}\left[P_c(i,j) > \theta\right], \quad \theta = 0.25$$

где $P_c(i,j)$ — softmax-вероятность класса $c$ для пикселя $(i,j)$.

### 1.2 Финальный набор из 5 признаков

Для каждого класса $c$ по массиву ratio-значений $\{r_1, r_2, \ldots, r_N\}$ по $N$ тайлам здания
вычисляются 5 агрегационных признаков:

| Признак | Формула | Физический смысл |
|---|---|---|
| `mean` | $\bar{r} = \frac{1}{N}\sum_{t=1}^{N} r_t$ | Средняя интенсивность дефекта по всему фасаду |
| `max` | $r_{\max} = \max_t r_t$ | Пиковая концентрация дефекта в худшем тайле |
| `prevalence` | $\frac{1}{N}\sum_t \mathbf{1}[r_t > \varepsilon],\ \varepsilon=0.05$ | Доля тайлов с видимым повреждением (охват) |
| `std` | $\sigma = \sqrt{\frac{1}{N}\sum_t (r_t - \bar{r})^2}$ | Неравномерность распределения дефекта по фасаду |
| `q75` | $\text{percentile}(r, 75)$ | Устойчивая оценка высокого уровня повреждения |

**Почему именно эти 5:** эксперименты с 8 признаками показали, что `skewness`, `concentration`
и `severity_coverage` имели отрицательные или нулевые веса во всех сессиях и увеличивали
дисперсию оценок. Переход с 8D на 5D снизил среднее стандартное отклонение Tau по фолдам
с 0.075 до 0.057 при незначительной потере в точности (−0.007 Tau).

Итоговый вектор признаков здания: **25D** (5 признаков × 5 классов).
Для каждой сессии используются 5 признаков соответствующего класса.

---

## 2. Принцип обучения: LR как ранкер

### 2.1 Формализация задачи

Дан датасет попарных сравнений $\mathcal{D} = \{(a_i, b_i, y_i)\}_{i=1}^{M}$, где:
- $a_i, b_i$ — векторы признаков двух зданий,
- $y_i = 1$, если здание $a_i$ повреждено сильнее, $y_i = 0$ — если $b_i$ хуже.
- Ничьи ($y = 0.5$) исключаются из обучения LR.

Цель: найти функцию оценки $f: \mathbb{R}^5 \to \mathbb{R}$ такую, что
$f(a) > f(b) \Leftrightarrow$ здание $a$ повреждено сильнее.

### 2.2 Вероятностная модель RankNet

RankNet моделирует вероятность того, что $a$ хуже $b$:

$$P(a \succ b) = \sigma\bigl(f(a) - f(b)\bigr) = \frac{1}{1 + e^{-(f(a) - f(b))}}$$

При линейной $f(\mathbf{x}) = \mathbf{w}^\top \mathbf{x}$ получаем:

$$P(a \succ b) = \sigma\bigl(\mathbf{w}^\top \mathbf{a} - \mathbf{w}^\top \mathbf{b}\bigr)
= \sigma\bigl(\mathbf{w}^\top (\mathbf{a} - \mathbf{b})\bigr)$$

### 2.3 Эквивалентность с Логистической Регрессией

Введём переменную $\mathbf{z}_i = \mathbf{a}_i - \mathbf{b}_i$. Тогда:

$$P(y_i = 1 \mid \mathbf{z}_i) = \sigma(\mathbf{w}^\top \mathbf{z}_i)$$

Это в точности **бинарная логистическая регрессия** с входом $\mathbf{z}_i$ и меткой $y_i$.
Функция потерь (log-loss) совпадает с BCE-лоссом RankNet:

$$\mathcal{L} = -\frac{1}{M} \sum_{i=1}^{M}
\bigl[y_i \log P_i + (1-y_i) \log(1-P_i)\bigr]$$

**Вывод:** Logistic Regression на разности признаков $\equiv$ линейный RankNet.
Если LR не уступает нелинейному MLP — зависимость в данных линейная.

### 2.4 Процедура обучения

```
Для каждого класса c ∈ {coating, cracks, masonry, moisture, vandalism}:

  1. Загрузить ranking_pairs_{c}.csv  (разметка аннотатора)
  2. Отфильтровать ничьи (winner == 2)
  3. KFold(n_splits=5, shuffle=True, random_state=42):
       a. Fit StandardScaler на зданиях из train-пар (не val!)
       b. Построить матрицу X = [vec_a - vec_b], y ∈ {0, 1}
       c. Fit LogisticRegression(C=0.1)
       d. Оценить Kendall Tau-b на val-фолде
  4. Обучить финальную модель на всех парах
```

**Важно:** `C=0.1` — усиленная L2-регуляризация. При 80–170 обучающих парах
и 5 признаках модель склонна к переобучению. Меньшее C = более сильный регуляризатор.

**Важно:** StandardScaler обучается только на **train-зданиях** внутри каждого фолда,
чтобы исключить утечку информации из валидационной выборки.

## 3. Результаты и интерпретация

### 3.1 Качество модели (5-Fold Cross-Validation)

| Класс | CV Tau-b | Std | Accuracy | Интерпретация |
|---|---|---|---|---|
| vandalism | **0.605** | ±0.059 | 0.823 | Граффити визуально однозначны, SegFormer детектирует хорошо |
| moisture\_bio\_damage | **0.536** | ±0.025 | 0.820 | Самый стабильный класс |
| coating\_deterioration | **0.526** | ±0.075 | 0.790 | Хорошее качество |
| masonry\_degradation | **0.493** | ±0.068 | 0.758 | Хорошее качество |
| **cracks** | **0.304** | ±0.132 | 0.678 | Слабый класс — следствие IoU=0.24 у SegFormer |
| **Mean** | **0.493** | ±0.066 | | |

**Интерпретация Kendall Tau-b:**
$$\tau_b = \frac{P - Q}{\sqrt{(P+Q+T_x)(P+Q+T_y)}}$$

где $P$ — число согласованных пар, $Q$ — несогласованных, $T_x, T_y$ — связанные ранги.
При Tau=0.5 модель правильно ранжирует 75% пар:
$$\text{Accuracy} = \frac{\tau_b + 1}{2}$$

### 3.2 Анализ весов

Финальные веса LR по классам (признаки: mean, max, prevalence, std, q75):

| Класс | mean | max | prevalence | **std** | q75 |
|---|---|---|---|---|---|
| coating | 0.177 | 0.221 | 0.405 | **0.402** | 0.207 |
| cracks | 0.224 | 0.303 | −0.103 | **0.324** | 0.102 |
| masonry | −0.004 | 0.094 | 0.484 | **0.429** | 0.010 |
| moisture | 0.141 | 0.336 | 0.501 | **0.451** | −0.069 |
| vandalism | 0.605 | 0.028 | 0.393 | **0.195** | 0.465 |

**Наблюдения:**
- `std` — единственный новый признак с стабильно высоким положительным весом.
  Высокое std означает, что дефект сконцентрирован в отдельных тайлах, а не размазан
  равномерно — аннотаторы воспринимают это как более серьёзное повреждение.
- `prevalence` доминирует в masonry и moisture: важна площадь охвата.
- `max` важен для moisture: аннотаторы реагируют на пиковые очаги влаги.
- `cracks`: отрицательный вес prevalence (−0.103) — следствие шумных признаков
  из-за низкого IoU сегментатора для этого класса.

### 3.3 Проблема класса cracks

Класс трещин демонстрирует низкое качество на всех уровнях:

| Уровень | Метрика | Значение |
|---|---|---|
| Сегментация | IoU | 0.24 |
| Ранжирование | CV Tau-b | 0.304 ± 0.132 |

Причина: тонкие трещины визуально похожи на `coating_deterioration` — SegFormer путает
эти два класса (31% пикселей трещин → background, 25% → coating). Шумные признаки
невозможно ранжировать надёжно.

## 4. Вычисление Damage Score и Global Damage Index

### 4.1 Scoring function

После обучения LR функция оценки для здания $\mathbf{x}$:

$$f_c(\mathbf{x}) = \mathbf{w}_c^\top \mathbf{x}_c$$

где $\mathbf{w}_c$ — вектор весов LR для класса $c$, $\mathbf{x}_c$ — нормализованный
(StandardScaler) вектор 5 признаков здания по классу $c$.

**Почему без intercept:** intercept $b$ обучался на разностях $\mathbf{a} - \mathbf{b}$
и отражает асимметрию в разметке пар (например, перекос в сторону "правое хуже").
При скоринге отдельного здания этот сдвиг не имеет смысла и исключается.

### 4.2 Якорная калибровка (Hybrid Anchor Calibration)

Сырые значения $f_c(\mathbf{x})$ — относительные баллы без фиксированного нуля.
Калибровка привязывает шкалу к физическому смыслу:

**Шаг 1 — Вычислить якорь (теоретически чистое здание):**

$$s_{\text{anchor}} = \mathbf{w}_c^\top \cdot \text{StandardScaler}(\mathbf{0})$$

Вектор $\mathbf{0} = [0, 0, 0, 0, 0]$ соответствует зданию без каких-либо повреждений
(SegFormer не нашёл ни одного пикселя дефекта). После нормализации через StandardScaler
этот вектор смещается относительно среднего обучающей выборки.

**Шаг 2 — Сдвиг:**

$$\tilde{s}_c(\mathbf{x}) = f_c(\mathbf{x}) - s_{\text{anchor}}$$

После этого здание без повреждений получает оценку $\approx 0$.

**Шаг 3 — Нормировка на P99:**

$$\hat{s}_c(\mathbf{x}) = \text{clip}\!\left(\frac{\tilde{s}_c(\mathbf{x})}{\text{P99}(\tilde{s}_c)},\ 0,\ 1\right)$$

P99 — 99-й перцентиль по всей выборке. Самое повреждённое здание в датасете
получает оценку ≈ 1.0. `clip` обрезает выбросы в $[0, 1]$.

**Пример для `coating_deterioration_score`:**

```
Здание A: coating_mean=0.12, max=0.31, prevalence=0.67, std=0.09, q75=0.22
→ Нормализация через scaler → vec_scaled
→ score_raw = vec_scaled @ w_coating          # например, 0.84
→ anchor_score = zeros_scaled @ w_coating     # например, 0.21
→ shifted = 0.84 - 0.21 = 0.63
→ P99 по всей выборке = 1.15
→ coating_deterioration_score = clip(0.63 / 1.15, 0, 1) = 0.55
```

### 4.3 Global Damage Index

Финальный индекс — среднее арифметическое по 5 классам:

$$\text{GDI}(\mathbf{x}) = \frac{1}{5} \sum_{c=1}^{5} \hat{s}_c(\mathbf{x})$$

Диапазон: $[0, 1]$.
- $0.0$ — здание без видимых повреждений
- $1.0$ — здание с повреждениями на уровне 99-го перцентиля выборки по всем классам

**Свойства индекса:**
- Монотонен относительно каждого класса в отдельности
- Устойчив к выбросам (P99 вместо max)
- Готов к подаче в GNN как нормализованный узловой признак $\in [0, 1]$

## 5. Итог и ограничения

### Что работает
- Парное ранжирование с LR даёт интерпретируемую линейную модель
- `std` как новый признак стабильно улучшает качество
- 5-Fold CV даёт надёжную оценку качества при малом датасете
- Mean CV Tau-b = **0.493** на 5 классах

### Ограничения
- **Класс cracks** ненадёжен (Tau=0.304, Std=0.132) — первопричина в IoU=0.24 сегментатора
- **Малый датасет**: 80–170 пар на сессию — нелинейные модели (RankNet MLP) не дают преимущества
- **Global Damage Index** — среднее по классам без взвешивания по серьёзности ущерба
- Якорная калибровка зависит от состава выборки: P99 смещается при добавлении новых зданий
