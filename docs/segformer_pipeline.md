# Пайплайн сегментации фасадов: SegFormer-b2 — от датасета до building_features

## Обзор

Задача: для каждого из 300 зданий получить числовой вектор признаков,
описывающий степень повреждения по 5 классам дефектов.

Пайплайн состоит из трёх стадий:
1. **Обучение SegFormer-b2** на размеченном датасете фасадов
2. **Инференс** — нарезка панорам на тайлы и сегментация
3. **Агрегация** — вычисление 40D вектора признаков здания

## 1. Датасет и разметка

### 1.1 Состав данных

| Стадия | Изображения |
|---|---|
| Начальная разметка (v1) | 206 |
| Финальное обучение (v2, `final_fit`) | 408 |

Из RGB-масок строятся индексные маски (`H × W`, dtype=`uint8`): каждый пиксель принимает

## 2. Выбор архитектуры: SegFormer-b2

### 2.1 Почему не U-Net

| Критерий | U-Net (CNN) | SegFormer (ViT) |
|---|---|---|
| Рецептивное поле | Локальное, растёт со слоями | Глобальное с первого блока (Self-Attention) |
| Артефакты на границах тайлов | Заметны — свёрточное ядро не видит контекст | Минимальны — токены кодируют всё изображение |
| Устойчивость к масштабному сдвигу | Слабая | Высокая (иерархический энкодер MiT) |
| Скорость инференса | Быстрая | Быстрая (сверхлегкий MLP-декодер) |

Панорамы зданий подаются в сеть как последовательность тайлов 800×800.
Без глобального контекста CNN оставляет «швы» на стыках — SegFormer этого лишён.

### 2.2 Архитектура SegFormer-b2

```
Энкодер: MiT-b2 (Mix Transformer)
  ├─ 4 иерархических стадии с Self-Attention
  ├─ Overlap Patch Embedding (stride < kernel → сохраняет локальные переходы)
  └─ Efficient Self-Attention: сокращение sequence length через Average Pooling

Декодер: All-MLP Head
  ├─ Проекция признаков всех 4 стадий в единое пространство
  ├─ Конкатенация и MLP → предсказание на 1/4 разрешения
  └─ Bilinear Upsample → оригинальный размер
```

Предобученные веса: `nvidia/mit-b2` (ImageNet-21k).

## 3. Обучение

### 3.1 Аугментации

Применяются библиотекой **albumentations**:

**Пространственные:**
- `RandomCrop` — случайный кроп до размера обучения
- `HorizontalFlip` — зеркальное отражение по горизонтали
- `RandomRotate90` — поворот на 0/90/180/270°

**Цветовые (на RGB изображения, маски не затрагиваются):**
- `ColorJitter` — случайное изменение яркости, контраста, насыщенности
- `HueSaturationValue` — сдвиг оттенка и насыщенности

Цель: повысить устойчивость к разным условиям съёмки (время суток, погода, угол камеры).

### 3.2 Компенсация дисбаланса классов

Фасады состоят преимущественно из фона. Дефектные классы занимают
малую долю пикселей. Без компенсации сеть игнорирует редкие классы.

**Метод: Median Frequency Balancing**

$$w_c = \frac{\text{median}(f_1, \ldots, f_C)}{f_c}$$

где $f_c$ — доля пикселей класса $c$ в обучающем датасете.

| Класс | Частота $f_c$ | Вес $w_c$ |
|---|---|---|
| background | высокая | **0.0622** |
| coating\_deterioration | средняя | **0.5067** |
| cracks | низкая | **6.6031** |
| masonry\_degradation | низкая | **4.8248** |
| moisture\_bio\_damage | средняя | **0.5578** |
| vandalism | очень низкая | **8.7833** |

Веса передаются в `CrossEntropyLoss(weight=class_weights, ignore_index=255)`.

### 3.3 Конфигурация обучения

| Параметр | Значение |
|---|---|
| Оптимизатор | AdamW |
| Learning rate | 6e-5 |
| LR scheduler | Cosine Annealing |
| Batch size | 4 |
| Gradient accumulation steps | 2 (эффективный batch = 8) |
| Эпохи | 50 |
| Лучшая модель | по `mean_iou` на val split |
| Метрика отбора | `eval_strategy="epoch"`, `load_best_model_at_end=True` |

Реализация через `Trainer` HuggingFace Transformers с кастомным `ClassWeightsTrainer`,
переопределяющим `compute_loss` для подстановки взвешенного лосса.

### 3.4 Качество сегментации (mIoU)

| Класс | IoU |
|---|---|
| coating\_deterioration | ~0.55–0.65 |
| masonry\_degradation | ~0.50–0.60 |
| moisture\_bio\_damage | ~0.55–0.65 |
| vandalism | ~0.60–0.70 |
| **cracks** | **~0.24** |
| **mean\_iou** | **~0.50–0.55** |

Класс `cracks` — слабое место сегментатора: тонкие трещины визуально похожи
на `coating_deterioration`. Примерно 31% пикселей трещин предсказываются как фон,
25% — как coating. Это ограничение напрямую ограничивает качество ранжирования.

## 4. Подготовка изображений к инференсу

### 4.1 Проблемы исходных панорам

Широкие панорамы, снятые с улицы, содержат:
- **Соседние здания** по краям кадра — сегментатор собирал бы дефекты с чужих фасадов
- **Масштабный сдвиг** — разная дистанция съёмки меняет размер трещины в пикселях
- **Изображения разного размера** — нельзя подавать в модель напрямую

### 4.2 Предобработка (три шага)

**Шаг 1 — Кадрирование (Cropping)**

Ручное или полуавтоматическое обрезание краёв кадра по границам целевого фасада.
Исключает соседние здания; обрезка сохраняет только пиксели целевого фасада.

**Шаг 2 — Масштабирование (Scaling)**

Приведение всех фотографий к единому контролируемому разрешению.
Устраняет масштабный сдвиг: один пиксель соответствует примерно одному
физическому размеру на поверхности фасада во всей выборке.

**Шаг 3 — Нарезка на тайлы (Tiling)**

Каждая масштабированная панорама нарезается на перекрывающиеся тайлы
$800 \times 800$ пикселей — размер совпадает с обучающей выборкой:

$$N_{\text{tiles}} = \left\lceil \frac{W - w}{s} + 1 \right\rceil \times \left\lceil \frac{H - h}{s} \right\rceil$$

где $w = h = 800$ — размер тайла, $s$ — шаг (stride < 800 обеспечивает перекрытие).

Имя файла тайла: `{building_name}_{row}_{col}.jpg` — позволяет восстановить принадлежность
тайла зданию при агрегации. Всего обрабатывается **6693 тайла** для 300 зданий.

Директория тайлов: `tiles_800x800_inference-300/`

## 5. Инференс SegFormer

### 5.1 Загрузка модели

```python
MODEL_DIR = "segformer-b2-final-fit-408"
model = SegformerForSemanticSegmentation.from_pretrained(MODEL_DIR)
processor = SegformerImageProcessor.from_pretrained(MODEL_DIR)
model.eval().to(device)
```

Модель `segformer-b2-final-fit-408` обучена на 408 изображениях (финальный fit).

### 5.2 Обработка одного тайла

```python
inputs = processor(images=image, return_tensors="pt").to(device)
with torch.no_grad():
    outputs = model(**inputs)

# Upsampling logits до исходного размера тайла
upsampled_logits = torch.nn.functional.interpolate(
    outputs.logits,
    size=image.size[::-1],
    mode="bilinear",
    align_corners=False
)

# Softmax → вероятности каждого класса для каждого пикселя
probs = torch.nn.functional.softmax(upsampled_logits, dim=1)
              .squeeze(0).cpu().numpy()  # shape: (6, H, W)
```

### 5.3 Вычисление ratio с мягким порогом

**Почему не argmax:**
Argmax присваивает пиксель классу с наибольшей вероятностью, даже если она 20%
против 19% у второго класса. Слабые, неуверенные детекции теряются.

**Softmax + порог $\theta$:**

$$\text{ratio}_{c,t} = \frac{1}{H \cdot W} \sum_{i,j} \mathbf{1}\left[P_c(i,j) > \theta\right],
\quad \theta = 0.25$$

При $\theta = 0.25$ пиксель засчитывается в класс $c$, если SegFormer уверен хотя бы
на 25%. Это восстанавливает слабые детекции, которые argmax пропускал,
и значительно улучшает заполнение гистограммы признаков.

```python
CONFIDENCE_THRESHOLD = 0.25
EPSILON = 0.05  # порог видимости для prevalence

for class_idx, class_name in enumerate(CLASS_NAMES[1:], start=1):
    class_probs = probs[class_idx]  # (H, W)
    tile_ratios[class_name] = float((class_probs > CONFIDENCE_THRESHOLD).mean())
```

## 6. Агрегация признаков здания (40D вектор)

### 6.1 Схема агрегации

После обработки всех тайлов здания получается массив ratio-значений
$\{r_1^c, r_2^c, \ldots, r_N^c\}$ — по одному на тайл, для каждого класса $c$.

Для каждого класса вычисляются **8 агрегационных признаков**:

| Признак | Формула | Физический смысл |
|---|---|---|
| `mean` | $\bar{r} = \frac{1}{N}\sum_t r_t$ | Средняя интенсивность дефекта по фасаду |
| `max` | $r_{\max} = \max_t r_t$ | Пиковая концентрация в худшем тайле |
| `prevalence` | $\frac{1}{N}\sum_t \mathbf{1}[r_t > \varepsilon],\ \varepsilon=0.05$ | Доля тайлов с видимым повреждением |
| `std` | $\sigma_r = \sqrt{\frac{1}{N}\sum_t(r_t - \bar{r})^2}$ | Неравномерность распределения дефекта |
| `q75` | $\text{percentile}(r, 75)$ | Устойчивая оценка высокого уровня |
| `skewness` | $\frac{\frac{1}{N}\sum_t(r_t-\bar{r})^3}{\sigma_r^3}$ | Асимметрия — перекос к редким тяжёлым очагам |
| `concentration` | $\frac{r_{\max}}{\bar{r} + \varepsilon}$ | Отношение пика к среднему — локализованность |
| `severity_coverage` | $\bar{r} \cdot \text{prevalence}$ | Комбинированная оценка: площадь × интенсивность |

**Итого: 8 признаков × 5 классов = 40D вектор**

Файл: `building_features_300_enriched.csv` (строки — здания, столбцы — признаки + `building_id`).

### 6.2 Граничный случай

Если здание не имеет ни одного тайла (`num_tiles == 0`):

```python
if num_tiles == 0:
    print(f"[SKIP] '{base_name}' — 0 тайлов")
    continue
```

Такое здание исключается из датасета. В нормальной ситуации каждое здание имеет ≥1 тайл.

### 6.3 Дополнительные производные признаки (enriched)

В ноутбуке 3 (`segformer-inference-analysis.ipynb`) вычисляются кросс-классовые признаки:

| Признак | Формула |
|---|---|
| `{cls}_log_mean` | $\log(1 + \text{mean}_c)$ |
| `{cls}_log_max` | $\log(1 + \text{max}_c)$ |
| `total_mean` | $\sum_c \text{mean}_c$ |
| `total_max` | $\max_c \text{max}_c$ |
| `num_damaged_classes` | $\sum_c \mathbf{1}[\text{mean}_c > 0]$ |
| `total_prevalence` | $\frac{1}{5}\sum_c \text{prevalence}_c$ |

### Ограничения

- **Класс `cracks`**: IoU=0.24 — трещины путаются с coating. Шумные ratio-значения
  ограничивают качество на всех последующих стадиях.
- **Порог $\theta=0.25$**: выбран эмпирически. Более низкий порог увеличивает recall
  слабых дефектов, но может вводить ложноположительные детекции.
- **Ручное кадрирование**: субъективность оператора вносит небольшую погрешность
  в границы фасада.
- **Масштабирование**: при сильно различающихся расстояниях съёмки пиксельный размер
  дефекта всё ещё варьируется. Полное устранение потребовало бы калибровки камеры.
