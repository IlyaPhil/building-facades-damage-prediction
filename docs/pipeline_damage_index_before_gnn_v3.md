# Пайплайн построения Damage Index (до стадии GNN) — v4 (Обновленный и исполненный)

Документ описывает полный пайплайн сбора данных, обучения моделей и калибровки данных, фактически выполненный в ходе подготовки признакового пространства для GNN.

## Ключевые методологические и академические решения (Обоснования)

1. **Выбор SegFormer вместо U-Net**
   * **Проблема:** Классические CNN (U-Net) обладают ограниченным локальным рецептивным полем и испытывают сложности с контекстом широких панорам. При нарезке изображений на тайлы (для инференса) сверточные сети оставляют заметные артефакты ("швы") на границах.
   * **Решение:** Архитектура SegFormer (Vision Transformer) благодаря механизму *Self-Attention* обладает глобальным рецептивным полем с первых слоев. Иерархический энкодер MiT делает модель устойчивой к масштабному сдвигу (scale variance). Сверхлегкий MLP-декодер обеспечивает более быстрый инференс огромного числа тайлов без падения качества (mIoU).
2. **Отказ от экспертных (ручных) формул взвешивания**
   * **Проблема:** Первоначальный подход с взвешенным суммированием сырых коэффициентов сегментации показал низкое соответствие человеческому восприятию (~60%). Причина в физической размерности: доля площади (`prevalence` и масштабный `mean`) математически подавляла пиковую интенсивность разрушений (`max`).
   * **Решение:** Переход к попарному ранжированию методами машинного обучения (Learning to Rank) с помощью сиамских нейросетей (RankNet).
3. **Разделение на 5 узких классов вместо 3 "суперкатегорий"**
   * Вместо смешанных групп (механические, биологические, антропогенные) были выделены 5 точечных категорий: `coating_deterioration`, `cracks`, `masonry_degradation`, `moisture_bio_damage`, `vandalism`. Итоговый вектор признаков был расширен до **15D**.
4. **Гибридная калибровка выходов (Semi-empirical Min-Max Scaling)**
   * **Проблема:** RankNet генерирует относительные баллы (без жесткой привязки чистого здания к нулю).
   * **Решение:** Выходы сети нормализуются: теоретическое идеальное здание `[0,0,0]` выступает якорем для `0.0`. Эмпирический максимум выборки (самые разрушенные руины в городе, P99) выступает пределом `1.0`. Будущие значения строго обрезаются `np.clip`, чтобы GNN работала со стандартизованным диапазоном.

# Фаза 1–3 — Сбор датасета, предобработка и инференс 

**Обучающая выборка и Аугментации:**
При обучении сегментатора применялись классические пространственные аугментации (случайные кропы, отражения) и цветовые искажения (ColorJitter) для повышения робастности модели к разным условиям уличной съемки и освещения.

**Подготовка данных для инференса:**
Сбор широких панорам сопряжен с рядом проблем: в один кадр часто попадают соседние здания, а также возникает геометрический «масштабный сдвиг» (из-за разной дистанции съемки размер трещины в пикселях варьируется от здания к зданию). Для решения проблем пайплайн инференса включал следующие преобразования:
1. **Кадрирование по целевому фасаду (Cropping):** Обрезка лишних смежных фасадов по краям кадра, чтобы сегментатор не собирал дефекты с чужих зданий.
2. **Масштабирование (Scaling):** Приведение всех фотографий к контролируемому оптическому разрешению.
3. **Нарезка на тайлы (Tiling):** Все панорамы перед подачей в нейросеть «разрезались» на перекрывающиеся тайлы. Это гарантировало подачу в сеть масштабов, идентичных обучающей выборке, без потери мелких деталей (трещин).

После сегментации всех тайлов для каждого здания были вычислены 3 метрики агрегации:
- `mean` — средняя доля пикселей поверхности, покрытая дефектом (усредение по всем тайлам конкретного здания).
- `max` — наихудшая концентрация дефекта в одном локальном тайле («пиковый урон»).
- `prevalence` — процент тайлов фасада, на которых дефект превышает пороговое значение видимости (оценка площади распространения).

Вектор признаков: **15D вектор** (3 агрегации × 5 классов).
Файл данных: `building_features_206.csv`

# Фаза 4 — Сбор датасета попарных сравнений (Разметка)

**Стратегия:**
Для независимого обучения 5 "голов" сети, ранжирование проводилось в 5 изолированных сессиях. В рамках каждой сессии отбирались только те фасады, где SegFormer обнаружил хотя бы >0.0% признаков повреждения (устранение нулевых пар).

**Валидация гипотезы ML:**
За счет сокращения входного пространства сети (подача только 3 специфичных признаков вместо всего 15D вектора), оказалось достаточно собрать **от 50 до 100 пар** на каждый класс. Компактное представление предотвращает переобучение на таких объемах.

# Фаза 5 — Обучение модели ранжирования (RankNet MLP)

## 5.1 Архитектура и подготовка
- **Нормализация:** Извлеченные 15D матрицы предварительно нормализованы (`StandardScaler`).
- **Сеть (RankNetHead):** Сиамская нейросеть `Linear(3 -> 8) -> ReLU -> Linear(8 -> 1)`. Легковесная конфигурация.
- **Прямой проход (Forward):** Инференс для вектора A, инференс для вектора B. Возвращаемое значение: `score_A - score_B`.

## 5.2 Обучение
Функция потерь: `BCEWithLogitsLoss`.
Таргеты конвертированы из ответов разметчика:
- Левое хуже (0) → `1.0`
- Правое хуже (1) → `0.0`
- Одинаково (2) → `0.5`

Модели обучаются отдельно в цикле, трекинг метрик (Loss, Accuracy) осуществляется через интеграцию с `WandB`.

# Фаза 6 — Оценка качества и Метрика Кендалла (Kendall's Tau)

В качестве метрик используются Pairwise Accuracy (Попарная точность) и Коэффициент ранговой корреляции Кендалла (Kendall's Tau).
Поскольку датасет состоит исключительно из попарных сравнений, RankNet делает бинарное предсказание (А больше Б). В таком сценарии Kendall's Tau имеет **прямую математическую связь с Accuracy**.

> Формула: `Kendall's Tau = (Число согласованных пар - Число несогласованных) / Общее число пар`
> Что эквивалентно: `Tau = 2 * Accuracy - 1`
> *То есть, при Accuracy сети 84%, Kendall's Tau = 0.68.*

# Фаза 7 — Получение итогового Global Damage Index

1. Обученные головы прогоняют через себя все 206 зданий (предъявляя по 3 нормализованных признака).
2. Выходы пропускаются через Sigmoid (возвращаются относительные оценки).
3. Производится **Гибридная Калибровка**:
   * Минимальный балл вычисляется искусственным прогоном вектора `[0,0,0]`.
   * Из всех оценок вычитается этот минимальный балл (чистые фасады обнуляются до 0.0).
   * Остаток делится на максимум самого разрушенного в датасете здания.
4. Оценки всех 5 классов усредняются (взвешиваются), формируя единый `Global_Damage_Index`.
5. Индекс полностью подготовлен к интеграции в GNN в качестве стандартизованной характеристики узла.
